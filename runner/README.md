# Bank-link runner

Раннер авто-оформления банковских ссылок через **Dolphin Anty + мобильный прокси**.

## Зачем он отдельный

Сайт живёт на Vercel (serverless) и **не может** достучаться до `localhost:3001`, где
Dolphin отдаёт браузер для автоматизации. Поэтому сайт только кладёт задачу в очередь
(`bank_link_jobs` в Supabase), а этот раннер — на машине с Dolphin — её забирает,
открывает форму в реальном браузере, заполняет, отправляет и достаёт «красивую» ссылку.

```
Сайт (Vercel)  ──insert──▶  bank_link_jobs (Supabase)  ◀──poll──  Раннер (эта папка)
                                    ▲                                   │
                                    └───────── result_link ◀───────────┘
```

## Требования

- **Node 18+**
- **Dolphin Anty** установлен и **запущен** на этой машине (окно можно свернуть).
  Local API включён (по умолчанию слушает `localhost:3001`).
- Мобильный прокси и токен Dolphin.

## Установка и запуск

```bash
cd runner
npm install
cp .env.example .env      # заполнить значения
npm start                 # node worker.js
```

Раннер будет крутиться и печатать лог по каждой задаче. Останавливается `Ctrl+C`.

Чтобы видеть, что происходит в браузере (отладка селекторов) — поставь `HEADLESS=0`.

## Перенос на VPS (позже)

Код переносимый. На VPS: поставить Dolphin (с GUI-окружением или в headless-режиме
согласно их доке), склонировать эту папку, задать другой `RUNNER_ID`, запустить под
`pm2`/`systemd`. Захват задач атомарный (`FOR UPDATE SKIP LOCKED`) — Mac и VPS можно
держать одновременно, одну задачу дважды не возьмут.

## Ссылка оформления (BANK_LINK_ALFA_URL)

Это НЕ env раннера, а **env сайта на Vercel**. Туда кладётся твоя клик-ссылка
(`https://trk.ppdu.ru/click/...`) — раннер открывает её, она сама редиректит на форму
`banks-market.com` со свежим click_id. Так каждая заявка = отдельный клик в трекере.

## Форма уже размечена

`automation.js` → `FORM_PROFILE` настроен под реальную форму Альфа-РКО (снята с живой
страницы):

- поля адресуются по порядку + сверяются подписи (у инпутов нет id/name);
- организация вводится **по ИНН** → автозаполняет название, ИНН и город;
- юр.адрес — автокомплит, выбирается дом по совпадению;
- подсказки — `[role="option"]`, телефон — по маске.

Осталось подтвердить на **первом реальном прогоне** (`HEADLESS=0`, реальный клиент):

- **кнопку результата** «Заполнить заявку на сайте банка» — код берёт её `href`, а если
  это редирект по клику — ловит новый URL. На живой странице-результате сверим и,
  если надо, поправим текст/селектор в `FORM_PROFILE.resultLinkTexts`.
- **payload профиля Dolphin** (`dolphin.js` → `createProfile`) — собран рабочий минимум
  fingerprint; если Remote API вернёт 4xx, он подскажет недостающее поле под твою версию.
